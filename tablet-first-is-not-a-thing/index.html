Tablet first is not really a thing, right?

I work for a large variety of companies and agencies and one of the things I keep hearing, is something they call Tablet First Design. Especially design agencies seem to sell this as a genuine approach. I first heard about Tablet First about 2 years ago, but it seems to gain more traction lately and I wonder what it means.

This Tablet First approach is derived from Mobile First, the title of Luke Wroblewski’s book for A Book Apart and the name of the principle he coined. Mobile First is a design approach where you start with designing for mobile (small screens) and utilizing the device’s capabilities. The reasoning behind this approach is that the contraints force you to focus on what’s necessary for the user to achieve his task. That way you get rid of all the unnecessary cruft that you normally see on websites.

Before Mobile First came along Progressive Enhancement was mostly a technical thing, but Luke took it a step further and applied these principles to design as well. Serve mobile users with the best version of the site for mobile and build upon this for devices with more capabilities.

What is tablet first?

When compared to smartphones, tablets have 3 important characteristics in common: touchscreens, sensors and portability. Just like for smartphones these characteristics are not binary; it’s not a matter of true/false and yes or no, each device has varying support levels. a Samsung Note is more portable than an iPad Mini, which is more portable than a Surface. The same goes for screens, from black-and-white resistive touchscreens to 4k screens. These traits are inherent to the web as a continuum.

So to serve tablets with a Tablet First design, we have to consider the unpredictable nature of the web and the best way to do that is by using progressive enhancement. Start with designing for the least capable devices, and build upon this when possible and necessary.

Optimizing for tablets does not mean optimizing for the iPad Air. A tablet user might just as well be an e-book reader with a black-and-white screen and a low refresh rate.

Is it the screen?

Tablets tend to have smaller screens than desktop computers (except sometimes they don’t). So it might be a layout that’s built for a 1024x768 resolution, which is the resolution for iPads, but that’s not Tablet First, that’s iPad First. Or 2005 First because not that long ago we were all designing for was the most common resolution for desktop computers. I don’t think screen size is a good criterium as long as nobody can tell me if this 6.4” Sony Experia is a phone or a tablet.

So maybe it’s optimizing for touchscreens.

But what does that mean? Big hit-areas for links and buttons to make it touch-friendly? You could easily call that basic usability. It was already recommended before the iPad was unveiled. If anything, I’d call it touch-friendly.

Ok, so it’s not about screens then.

Sensors then?

So it might be about GPS, accelerometers, luminosity sensors and all these other cool things that are built into smartphones and tablets. As we’ve seen before the web is an unpredictable medium. You can’t rely on these sensors to be present, so you need to have a fallback, or rather enhance when they are present.

I don’t see this being any different for tablets than it is for mobile or even desktop.

It must be the portability then.

The portability of tablets effects its usage in 2 ways. You can 1) take it with you so you can use the GPS for location-based functionality and 2) you can hold it in your hands which can be used to create a ”submersive experience”.

The first effect is already discussed in the previous section. The second one is not that different from phones or dektops. If you disagree I advise you to look around in public transportation, most people are looking at their phones. How’s that for submersive?

So what’s left?

One of the biggest advantages of Mobile First are the constraints. They force you to focus on the actual content and the primary goal of a page. Just like Stephen Hay mentions in his book Responsive Design Workflow ’start designing from the content out’.

Mobile First is not about layout, but it’s about designing Content First and progressive enhancement.

I don’t see how Tablet First adds anything to that approach. If it’s about designing for smaller than 1600x1200 and large hit-areas, we already have a name for that:

It’s called web design.
